How do we use your Google account?

We ask for a single permission: View and manage Google Drive files that you have opened or created with this app. This permission lets us create files in your Drive, and update those files later. Also, if in the future we provide a way for you to directly upload Pyret files and load them into code.pyret.org from your Drive, you can explicitly grant permission for this application to view and edit certain files.

We use client-only authorization (if you're really interested, we use APIs similar to the ones described in https://developers.google.com/drive/web/quickstart/quickstart-js), so this application can only access your data from open pages on our site. We don't save any access tokens to our server.

You can always go to https://security.google.com/settings/security/permissions?pli=1 and immediately remove all access that we have to your account. Your programs and code.pyret.org folder will not be deleted if you do so, but you can always manually delete the folder and its contents yourself.

Where are things stored in my Drive?

We make a single folder in your Google Drive, called code.pyret.org, and store all of your programs there, with the names you choose for them.

We can only see files that we created, so simply moving another file into your code.pyret.org collection wonâ€™t cause it to show up in our interface: you need to create the file manually through our editor.

The listing of programs in this application only looks for the code.pyret.org folder and its contents, so if you rename or remove that folder, a new, empty one will be created for you on your next login.
